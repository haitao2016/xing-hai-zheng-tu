## Phase 5: 代码重构 (Code Health) 🏗️

**目标**: 解决 Core.lua 和 Render.lua 超负荷问题，提升可维护性

### 5.1 Core.lua 拆分

将 2300 行拆为 4 个职责清晰的模块：

```
scripts/game/
├── Core.lua         → 瘦身至 ~400 行（协调层 + newGame + update 主循环）
├── PlayerCtrl.lua   → ~300 行（玩家移动/射击/受伤/武器切换）
├── EnemyAI.lua      → ~600 行（敌人生成/AI行为/Boss逻辑）
├── Combat.lua       → ~400 行（子弹更新/碰撞检测/伤害计算/遗物效果）
└── World.lua        → ~300 行（星体生成/资源拾取/中继站/事件/波次）
```

### 5.2 Render.lua 拆分

将 2290 行拆为 3 个渲染层：

```
scripts/game/
├── Render.lua       → ~200 行（入口调度 + 颜色定义 + 工具函数）
├── RenderWorld.lua  → ~700 行（星图/玩家/敌人/子弹/特效/粒子）
├── RenderUI.lua     → ~700 行（HUD/科技树/GameOver/菜单/排行榜/统计）
└── RenderFX.lua     → ~400 行（Boss特效/慢动作/事件弹窗/教程/成就弹窗）
```

### 5.3 接口一致性审计

- 统一 toast 字段名（`text` vs `msg`）
- 统一 pickup 字段名（`kind` vs `type`）
- 统一粒子颜色格式（`color={r,g,b}` vs `r,g,b` 独立字段）
- 检查 `Core.checkQuests` 是否在 update 中调用

**预期收益**: 每个文件 < 800 行，单元可独立理解和修改

## Phase 6: 游戏手感打磨 (Game Feel) ✨

**目标**: 让已有机制的反馈更加"有感"

### 6.1 屏幕震动增强

当前已有 `shakeTime/shakeIntensity`，需扩展触发场景：

| 触发事件 | 强度 | 持续 |
|---------|------|------|
| Boss 出场 | 6.0 | 0.8s |
| Boss 击杀 | 10.0 | 1.2s |
| 玩家受到大额伤害(>30) | 4.0 | 0.3s |
| 导弹爆炸 | 3.0 | 0.2s |
| 隐藏Boss技能命中 | 8.0 | 0.5s |

### 6.2 击杀反馈强化

- **击杀冻结帧**: 精英/Boss 击杀时暂停 2-3 帧（用现有 slowmo 系统）
- **爆炸粒子分级**: 普通敌人 5 粒子 → 精英 12 粒子+闪白 → Boss 30 粒子+全屏白闪
- **音效层级**: 普通击杀(轻响) / 精英(金属碎裂) / Boss(爆炸+回响)

### 6.3 运动拖尾 (Motion Trail)

- 玩家高速移动时引擎尾焰拉长
- 导弹飞行轨迹烟雾
- 激光蓄力阶段能量聚集粒子

### 6.4 UI 微动效

- Combo 数字跳动缩放（已有 displayTimer，增加弹性动画）
- 受伤时 HP 条抖动
- 资源获取时对应图标 pulse 效果
- 按钮 hover 高亮（菜单已有基础，优化手感）

## Phase 7: 内容扩展 (Content) 🎮

**目标**: 增加重复可玩性

### 7.1 第二武器栏 + 新武器

| 武器 | 类型 | 特点 |
|------|------|------|
| 散射炮 | 近程 | 3发扇形，近距离高伤害 |
| 回旋镖 | 中程 | 投出后返回，双重判定 |
| 等离子地雷 | 布置型 | 放置后延迟爆炸，范围伤害 |

- 按 `Tab` 切换主/副武器
- 科技树新增副武器分支

### 7.2 无尽模式 (Endless Mode)

- 取消赛季天数上限，无限推进
- 难度无限递增，每 10 天出一轮 Boss
- 单独排行榜（按存活天数+分数排名）
- 从菜单新增入口按钮

### 7.3 星际事件扩展

新增 4 种随机事件：

| 事件 | 效果 | 持续 |
|------|------|------|
| 时空裂缝 | 随机传送 + 大量资源 | 瞬发 |
| 太阳风暴 | 全屏持续伤害，必须躲避在小行星后 | 8s |
| 商人舰队 | 出现友方商人，可消耗资源购买遗物 | 15s |
| 虫洞 | 传送到隐藏区域，高风险高收益 | 20s |

### 7.4 更多遗物 (扩至 15 种)

新增 5 个遗物：

| ID | 名称 | 效果 |
|----|------|------|
| r_multishot | 多重射击 | 每次射击额外发射1发(±15°) |
| r_slowfield | 减速力场 | 周围敌人减速 30% |
| r_lucky | 幸运星 | 掉落率 +25%，暴击率 +10% |
| r_overclock | 超频核心 | 射速 +40%，但受伤 +20% |

## Phase 8: 持久化与社交 (Meta) 🏆

- 永久统计写入文件（当前仅内存）
- 成就解锁状态持久化

## Phase 10: 最终润色 (Final Polish) 🎯

### 10.1 新手引导优化

- 当前教程 5 步过于简单，扩展为 8 步
- 添加指引箭头（指向目标位置）
- 首次遇到新机制时弹出简要说明

### 10.2 难度曲线调优

- 前 3 天设为安全期（敌人少/弱）
- Day 5 开始引入精英

### 10.3 性能优化

- 粒子数量上限（当前无限制）
- 屏幕外实体跳过渲染
- 敌人 AI 分帧更新（远处敌人每 2 帧更新一次）

### 10.4 多语言准备

- 所有硬编码中文提取为常量表
- 为未来 i18n 留好接口

## 建议实施顺序

```
Phase 5 (重构)     ←── 最高优先级，后续所有改动基于此
  ↓
Phase 6 (手感)     ←── 快速提升体验质感
  ↓
Phase 8.1 (存档)   ←── 解决数据丢失问题
  ↓
Phase 7 (内容)     ←── 核心可玩性
  ↓
Phase 9 (音频)     ←── 沉浸感
  ↓
Phase 8.2-8.4      ←── 社交留存
  ↓
Phase 10 (润色)    ←── 最终上线前
```

**关键决策点**:
- Phase 5 重构是**阻塞项**——Core.lua/Render.lua 各 2300 行，继续堆积功能会导致维护崩溃
- Phase 6 手感打磨投入少回报大，建议紧跟重构之后
- Phase 7 内容和 Phase 9 音频可并行推进

## 技术风险

| 风险 | 影响 | 缓解 |
|------|------|------|
| Core.lua 拆分导致循环依赖 | 编译失败 | 提前设计模块接口，state 对象作为唯一共享数据 |
| 粒子/实体过多导致帧率下降 | 卡顿 | Phase 10.3 设置上限，分帧更新 |
| 存档格式变更兼容性 | 旧存档丢失 | 存档版本号 + 迁移函数 |
| 每日挑战种子跨时区不一致 | 排行不公平 | 使用 UTC 日期作为种子 |
