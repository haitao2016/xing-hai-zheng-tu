## Phase 11: 新游戏模式 (New Game Modes) ✅ 已完成

### 11.1 限时挑战模式 (Time Attack)

- 60秒限时生存，击杀越多分数越高
- 特殊规则：连击倍率翻倍，无科技升级
- 单独排行榜，记录最高击杀数

### 11.2 弹幕生存模式 (Bullet Hell)

- 大量敌方弹幕，考验走位能力
- 敌人不攻击，只发射弹幕
- 生存时间越长分数越高，奖励特殊皮肤

### 11.3 Boss Rush 模式

- 连续挑战所有Boss（含隐藏Boss）
- 每击败一个Boss恢复部分HP/护盾
- 记录最快通关时间

## Phase 12: 角色与剧情系统 (Story & Characters) 📖

**目标**: 增加游戏叙事深度，赋予战斗意义

### 12.1 角色故事线

- 3个阵营各有独立背景故事
- 游戏中通过事件/对话揭示剧情
- 完成特定条件解锁隐藏剧情

### 12.2 对话系统

- 商人舰队事件加入对话选项
- Boss出场时播放语音/文字介绍
- 完成赛季后解锁结局动画

### 12.3 世界观扩展

- 新增"星海编年史"收集系统
- 通过游戏内探索解锁背景资料
- 记录在主菜单的"资料库"中

## Phase 13: 深度内容扩展 (Deep Content) ✅ 已完成

**目标**: 增加长期可玩性，提供更多策略选择

### 13.1 新敌人类型 (新增4种)

| 敌人 | 特点 |
|------|------|
| 虚空吞噬者 | 吸收子弹转化为护盾 |
| 电磁干扰器 | 释放EMP脉冲，使玩家武器短暂失效 |
| 时空扭曲者 | 可以瞬移到玩家背后 |
| 量子分裂体 | 被击杀后分裂为2个小敌人 |

### 13.2 新Boss (新增2种)

| Boss | 阶段特点 |
|------|----------|
| 星际仲裁者 | 远程激光阵 + 召唤仲裁骑士 |
| 深渊巨口 | 吞噬小行星恢复HP + 全屏黑洞吸附 |

### 13.3 新遗物 (扩至25种)

新增 10 个遗物：

| ID | 名称 | 效果 |
|----|------|------|
| r_shield_charge | 护盾充能 | 移动时护盾缓慢恢复 |
| r_hp_to_shield | 生命转化 | HP低于50%时自动转化为护盾 |
| r_damage_reflect | 伤害反射 | 受到攻击时反弹20%伤害 |
| r_energy_drain | 能量汲取 | 攻击敌人时恢复少量能量 |
| r_time_warp | 时间扭曲 | 击杀敌人时概率触发短暂慢动作 |
| r_bullet_homing | 子弹追踪 | 普通子弹有50%概率追踪敌人 |
| r_shockwave | 冲击波 | 每5秒释放一次周围冲击波 |
| r_stealth | 隐形装置 | 静止2秒后进入隐形状态 |
| r_bonus_dmg | 弱点打击 | 攻击敌人背后伤害+50% |
| r_resource_magnet | 资源磁吸 | 资源拾取范围翻倍 |

### 13.4 新事件 (新增4种)

| 事件 | 效果 |
|------|------|

## Phase 15: UI/UX 升级 (UI/UX Upgrade) ✅ 已完成

**目标**: 提升界面美观度和操作便捷性

### 15.1 设置菜单

- 音量设置（主音量/BGM/音效独立调节）
- 画面设置（分辨率/全屏/帧率限制）
- 操作设置（自定义快捷键）
- 震动设置（开关/强度调节）

### 15.2 小地图系统

- 显示周围敌人/Boss位置
- 标记中继站和资源点
- 缩放控制

### 15.3 战斗信息面板

- 实时显示当前敌人数量/类型
- Boss血条显示（含阶段进度）
- 当前生效的增益/减益效果

### 15.4 移动端适配

- 虚拟摇杆控制移动
- 触摸瞄准/射击
- 手势操作（滑动切换武器等）

## Phase 16: Mod支持与扩展性 (Mod Support) 🔧

**目标**: 开放创作接口，支持社区内容

### 16.1 数据驱动架构

- 将敌人/Boss/遗物/事件配置完全数据化
- 提供 JSON 配置文件模板
- 游戏启动时自动加载 mod 配置

### 16.2 Mod 加载器

- 支持从指定目录加载 mod
- Mod 优先级管理（官方 > 社区）
- 冲突检测和警告

### 16.3 脚本扩展接口

- 提供 Lua API 扩展点
- 允许 mod 添加自定义事件/敌人行为
- 提供文档和示例

---

## Phase 17: 性能与稳定性优化 (Performance) ⚡

**目标**: 确保高负载场景下流畅运行

### 17.1 渲染优化

- 小行星/粒子使用批处理渲染
- 实现纹理图集减少 draw call
- 添加 LOD 系统（远距离物体简化）

### 17.2 物理优化

- 碰撞检测空间分区（QuadTree）
- 子弹生命周期管理优化
- 减少不必要的距离计算

### 17.3 内存管理

- 资源按需加载/释放
- 对象池优化（敌人/子弹/粒子）
- 内存泄漏检测

## 建议实施顺序

```
Phase 11 (新模式)   ←── 快速扩展玩法，吸引新玩家
  ↓
Phase 13 (内容)     ←── 核心可玩性提升，延长游戏时间
  ↓
Phase 15 (UI/UX)    ←── 提升体验质感
  ↓
Phase 14 (社交)     ←── 社区生态建设
  ↓
Phase 12 (剧情)     ←── 叙事深度，赋予意义
  ↓
Phase 16 (Mod支持)  ←── 长期扩展性
  ↓
Phase 17 (优化)     ←── 最终打磨
```

**关键决策点**:
- Phase 11 和 Phase 13 是核心内容，应优先推进
- Phase 16 Mod支持需要在数据结构稳定后再实施
- Phase 17 优化贯穿始终，定期进行性能测试

## 技术风险

| 风险 | 影响 | 缓解 |
|------|------|------|
| 新模式导致代码复杂度增加 | 维护困难 | 保持模块独立性，每个模式独立目录 |
| 数据量过大影响加载速度 | 卡顿 | 实现资源按需加载 |
| Mod系统兼容性问题 | 游戏崩溃 | 严格的配置校验和沙盒环境 |
| 多人社交功能服务器依赖 | 运维成本 | 优先使用现有排行榜基础设施 |
