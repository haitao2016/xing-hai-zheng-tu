# 星海征途 - v1.0.0 至 v2.0.0 开发计划

> 基于 v0.8.0 完整版本，面向正式发布和长期运营的路线图
> 已完成: Phase 1~17 (v0.1.0 → v0.8.0 全部核心功能)
> 本计划定义 Phase 18~27，覆盖 v0.9.0 → v1.0.0 → v2.0.0

---

## 当前版本诊断

| 维度 | 状态 | 关键缺口 |
|------|------|----------|
| 核心循环 | ✅ 稳定 | 深度不足，敌人 AI 模式单一 |
| 内容量 | ⚠ 基本够用 | 遗物/事件虽多但互作用弱 |
| UI/UX | ⚠ 基础完整 | 移动端无触控、动画不足 |
| 性能 | ✅ 健康 | 缺少动态 LOD/批处理 |
| 社交 | 🛠 仅有排行 | 无好友/组队系统 |
| 剧情 | ⚠ 雏形存在 | 没有战役关卡序列 |
| 跨平台 | 🛠 PC 优先 | 无触控/手柄支持 |
| Mod 生态 | ⚠ 接口已存在 | 无实际加载器 |

### 模块健康矩阵

| 模块 | 文件 | 健康度 | 主要问题 |
|------|------|--------|----------|
| 核心编排 | Core.lua | ✅ A | 新增系统过多，需拆分 state 结构 |
| 玩家控制 | PlayerCtrl.lua | ✅ A | 无输入抽象层（PC 键鼠硬编码） |
| 敌人 AI | EnemyAI.lua | ⚠ B | 行为模式有限，缺少精英专属 AI |
| 战斗 | Combat.lua | ⚠ B | 碰撞检测仍可优化 |
| 世界 | World.lua | ⚠ B | 波次系统过于简单 |
| 渲染 | RenderWorld.lua | ⚠ B | 着色器单一，无粒子批处理 |
| UI | RenderUI.lua | ⚠ B | HUD 信息超载，缺少动画 |
| 特效 | RenderFX.lua | ✅ A | 状态良好 |
| 系统 | Systems.lua | ⚠ B | 成就/科技/遗物系统可拆分 |
| 数据 | Data.lua | ⚠ B | 配置集中，可抽离 JSON |
| 音频 | Audio.lua | ✅ A | 状态良好 |
| 存档 | SaveSystem.lua | ⚠ B | 单槽位，无版本化管理 |
| 排行 | Leaderboard.lua | 🛠 C | 基础功能，需对接云端 |
| 主入口 | main.lua | ⚠ B | 状态机可用，但输入逻辑过散 |

## Phase 18: 战役与剧情模式（Campaign & Story Mode）

**目标**: 将"赛季模式"升级为结构化战役，赋予玩家清晰的叙事与目标推进感

### 18.1 战役关卡结构

- 将现有"赛季"重新设计为 **多章节战役**
- 每章 = 10 天（Day 1~10），对应 1 个 Boss
- 章节主题示例:
  - 第一章 · 边境星域 · 敌人: drone / aberration · Boss: 湍流尸骸
  - 第二章 · 科技禁区 · 敌人: guard / summoner · Boss: （新）数据核心
  - 第三章 · 虚空裂隙 · 敌人: 全部 + 虚空精英 · Boss: 隐藏Boss
- 每章节专属 BGM / 背景主题色

**实现要点**:
```
在 Data.lua 新增 CAMPAIGN = {
  chapters = {
    { id = 1, name = "边境星域", bgColor = {20,30,60},
      days = 10, boss = "turbulence", unlockKey = nil },
    ...
  }
}
在 Core.lua 新增 state.chapterIndex / chapterDay 状态
```

### 18.2 任务系统扩展（支线任务）

当前任务仅为触发 Boss 的"主线"。新增支线任务:

| 任务类型 | 示例 | 奖励 |
|----------|------|------|
| 击杀计数 | 击杀 20 个 drone | 蓝图 +3 |
| 收集 | 采集 50 单位金属 | 科技折扣券 |
| 生存 | 在单日不掉盾生存 60 秒 | 稀有遗物 |
| Boss 速杀 | 30 秒内击杀 Boss | 专属称号 |
| 建造 | 建造 3 座中继站 | 远古钥匙 +1 |

**实现要点**:
- Data.lua 新增 QUESTS_SIDE 表
- Core.lua 扩展 state.activeQuests / state.completedQuests
- RenderUI.lua 新增任务面板（右侧浮窗）

### 18.3 对话系统增强

- 每章开始播放剧情对话（类似 Boss 对话）
- 新增 NPC 角色: 商盟联络员 / 战团指挥官 / 学者会研究员
- 对话可选分支影响奖励（简易选择系统）

**实现要点**:
- Data.lua 新增 NPC_DIALOG = { merchant = {...}, warband = {...} }
- RenderUI.lua 新增 drawDialogBox() 对话框组件
- 对话状态由 state.dialogue = { active, npc, currentLine, choices } 管理

### 18.4 关卡目标多样化

当前每关=生存一天。改为混合目标:

- **生存目标**: 存活指定时间
- **击杀目标**: 在时间内击杀 N 个敌人
- **探索目标**: 靠近地图边缘的信号点
- **保护目标**: 保护中继站不被摧毁（新机制）
- **特殊**: Boss 关

## Phase 19: 难度层级与玩家成长（Difficulty & Progression）

**目标**: 让不同水平的玩家都能找到乐趣，同时提供长期成长目标

### 19.1 四档难度系统

| 难度 | 敌人 HP | 敌人伤害 | 掉落率 | 得分倍率 | 解锁条件 |
|------|---------|----------|--------|----------|----------|
| 新手 | 0.6x | 0.5x | 1.5x | 0.5x | 默认 |
| 标准 | 1.0x | 1.0x | 1.0x | 1.0x | 默认 |
| 困难 | 1.5x | 1.3x | 1.2x | 2.0x | 通关标准 |
| 虚空 | 2.0x | 1.8x | 1.5x | 4.0x | 通关困难 + 击杀隐藏Boss |

**实现要点**:
- Data.lua 新增 DIFFICULTY_LEVELS
- Core.lua 在 newState() 接收 difficulty 参数
- EnemyAI.spawnEnemy 应用 hpMul / dmgMul
- 难度影响 diffScale 的初始值

### 19.2 永久升级（Meta Progression）

在游戏外（主菜单）提供永久升级系统，用累计蓝图购买:

| 永久升级 | 效果 | 消耗 |
|----------|------|------|
| 核心加固 | 初始 HP +20 | 50 蓝图 |
| 引擎调校 | 初始速度 +10% | 40 蓝图 |
| 武器校准 | 初始伤害 +10% | 60 蓝图 |
| 护盾过载 | 初始护盾 +30 | 45 蓝图 |
| 幸运之神 | 掉落率 +10% | 80 蓝图 |
| 战术雷达 | 小地图范围 +20% | 30 蓝图 |

**实现要点**:
- SaveSystem.lua 增加 metaUpgrades 字段
- Core.lua 在 newGame() 时将永久升级应用到初始 stats
- main.lua 在 STATE_MENU 新增"升级面板"

### 19.3 解锁系统

完成特殊条件解锁新功能:

- 通关战役 → 解锁无尽模式（已存在）
- 困难通关 → 解锁"虚空模式"（全新，极高难度）
- 集齐 25 遗物 → 解锁"遗物随机"启动选项
- 全成就 → 解锁"传奇皮肤"

### 19.4 技能槽位扩展

当前遗物 3 槽。通过永久升级解锁至 5 槽:

- 默认: 3 槽
- meta 升级 +1 槽（4 槽）
- 全成就奖励 +1 槽（5 槽）

## Phase 20: 战斗系统深度化（Combat Deepening）

**目标**: 让战斗从"扫射"升级为有策略、有变化、有节奏感的体验

### 20.1 敌人行为模式扩展

新增 3 种精英敌人专属行为:

| 敌人 | 新行为 |
|------|--------|
| 盾卫 | 旋转能量盾，只能从背后命中（与 r_bonus_dmg 遗物联动） |
| 自爆兵 | 低血量时冲向玩家（已有，需增强视觉预警） |
| 召唤者 | 每 15 秒召唤增援（已有，需扩召唤物种类） |

**新增敌人 2 种**:

| 敌人 | 特点 | 章节 |
|------|------|------|
| 相位潜行者 | 周期性消失/出现在不同位置 | 第二章 |
| 能量吸取者 | 持续吸取玩家护盾/能量 | 第三章 |

### 20.2 玩家技能系统（主动技能）

当前只有激光/导弹（被动/冷却）。新增主动技能槽:

| 技能 | 效果 | 冷却 | 解锁条件 |
|------|------|------|----------|
| 冲刺闪避 | 短暂无敌 + 快速位移 | 8s | 默认 |
| 能量爆发 | 360 度冲击波 + 2秒无敌 | 20s | 购买科技 |
| 时间减速 | 全场敌人速度 50% 持续 4 秒 | 30s | 购买科技 |
| 护盾充能 | 立即恢复 50% 护盾 | 25s | 默认解锁 |
| 轨道打击 | 标记目标区域 3 秒后 3x 激光齐射 | 45s | 购买科技 |

**实现要点**:
- Combat.lua 新增 playerSkills 表（{id, cd, maxCd}）
- PlayerCtrl.lua 处理技能输入（数字键 1/2/3/4）
- RenderUI.lua 底部 HUD 技能图标

### 20.3 连击奖励深度化

当前 combo → 里程碑奖励。扩展为**连击等级（Combo Rank）**:

| 连击 | 等级 | 加成 |
|------|------|------|
| 0~4 | 无 | 基础伤害 1.0x |
| 5~9 | C | 伤害 1.1x + 视觉脉冲 |
| 10~19 | B | 伤害 1.25x + 屏幕震动增强 |
| 20~29 | A | 伤害 1.5x + 子弹拖尾增强 |
| 30~49 | S | 伤害 2.0x + 金色粒子光环 |
| 50+ | SSS | 伤害 3.0x + "完美击杀"特效 |

**实现要点**:
- Systems.lua 新增 comboRank 计算函数
- RenderUI.lua / RenderFX.lua 根据 comboRank 调整视觉效果强度

### 20.4 武器多样性（第二武器完整化）

当前有散射炮/回旋镖/地雷，但:
- 每次只能持有一种第二武器
- 需扩展武器切换机制（按 Q 键切换）
- 每种武器独立升级线（伤害/范围/冷却）

**实现要点**:
- Core.lua state 扩展 secondaryWeapons = { spread=true, boomerang=true, mine=false, current="spread" }
- Combat.lua fireSecondary() 根据 current 发射不同武器
- Data.lua 为每种武器设计升级链（如 spread_dmg_2 / spread_range_3）

### 20.5 敌弹模式多样化

| 模式 | 视觉 | 用途 |
|------|------|------|
| 普通直线弹 | 小圆圈 | 常规敌人 |
| 追踪弹 | 带尾迹 | 精英敌人 |
| 扩散弹 | 开花状 | Boss 阶段 |
| 螺旋弹 | 螺旋射线 | 新 Boss "数据核心" |
| 激光线 | 预警线 + 持续激光 | 新 Boss "深渊巨口" 技能 |

## Phase 21: 世界与探索系统（World & Exploration）

**目标**: 将战场从"随机场地"升级为有结构感的星域

### 21.1 区域化星域地图

将游戏世界划分为 4 个区域:

| 区域 | 主题色 | 敌人倾向 | 资源倾向 |
|------|--------|----------|----------|
| 边境星域 | 深蓝/白 | Drone, Guard | 金属多 |
| 科技禁区 | 青绿 | Summoner, Aberration | 能量多 |
| 虚空裂隙 | 紫黑 | 虚空系 | 蓝图多 |
| 核心战区 | 金红 | BOSS 区域 | 综合 |

**实现要点**:
- Data.lua 新增 ZONES = { ... }
- RenderWorld.lua 背景色根据当前区域动态调整
- 玩家可通过"空间传送门"事件在区域间切换

### 21.2 可发现的神秘地点

地图随机生成 3~5 个特殊地点:

| 地点 | 效果 |
|------|------|
| 古代废墟 | 停留 5 秒获得随机遗物（稀有） |
| 能量源 | 持续恢复护盾/能量，但吸引大量敌人 |
| 商人舰队 | 用资源购买遗物/解锁科技（已存在，可升级） |
| 废弃中继站 | 立即建造一座中继站（免费） |
| 时空裂缝 | 随机时间/随机传送（与新事件联动） |

### 21.3 波次系统升级

当前波次 = 随机敌人组合。改为**有节奏的波次**:

- 开场波: 少量敌人，热身
- 战斗波: 中等规模 + 混合类型
- 精英波: 每 5 天一次，精英敌人为主
- Boss 波: 每章最后一天
- 突袭波: 随机触发的高压力波（视觉警报 + 大量敌人）

**实现要点**:
- World.lua 重写 spawnWave 逻辑
- Core.lua 在 state.waveType 记录当前波次类型
- RenderUI.lua 波次开始时显示大字提示（如 "精英波来袭！"）

## Phase 22: UI/UX 重制（UI/UX Overhaul）

**目标**: 将现有"可玩"的 UI 升级为"易用/好看/有品牌感"

### 22.1 HUD 重构

当前问题:
- 信息过多，屏幕拥挤
- 关键信息（当前波次/距离 Boss 时间）不够突出

重构方向:

| 区域 | 内容 | 设计 |
|------|------|------|
| 左上 | HP/护盾/能量 | 水平进度条 + 数字 |
| 左下 | 第二武器/技能 | 圆角图标 + 冷却环 |
| 右上 | Day/得分/时间 | 大号字体 + 小图标 |
| 右下 | 小地图 | 圆形地图（替换方形） |
| 顶部中 | 社区挑战进度 | 小条形 + 图标 |
| 右中 | 任务/目标 | 可折叠面板 |
| 屏幕中央 | 连击数 | 仅在 combo>0 显示 |

### 22.2 主菜单重设计

- 简洁的星空背景（当前可增强动画）
- 大按钮卡片式设计
- 右侧展示"今日挑战/上周排行"
- 左下版本信息 + 同步按钮

### 22.3 科技树界面升级

当前科技树为列表式。改为**可视化节点图**:
- 节点/连线形式展示
- 高亮已解锁/可解锁/不可解锁
- 选中节点显示详情卡片（消耗/效果/前置条件）

### 22.4 过渡动画系统

- 菜单→游戏：淡入淡出（当前存在，可增强）
- 游戏→死亡：慢动作 + 红色晕染（已有 shake，可增加）
- 科技升级：节点闪光动画
- 新物品拾取：从屏幕边缘飞入图标 + 脉冲

### 22.5 提示与引导

- 首次拾取新遗物 → 弹出"遗物图鉴"条目
- 首次遇到新敌人 → 弹出"敌人档案"
- 首次解锁新科技 → 简要提示"已解锁新能力"

## Phase 23: 音频与视觉呈现（Audio & Visual Polish）

**目标**: 建立独特的听觉与视觉语言，让玩家"一听就知道是星海征途"

### 23.1 动态音频（扩展当前）

当前已有 BGM 战斗/巡航切换。扩展:

| 触发 | 音效 |
|------|------|
| 低血量（<30%） | 心跳声 + BGM 切换紧张版 |
| 敌人接近屏幕外 | 3D 音频方向提示 |
| Boss 登场前 5 秒 | 低频脉冲预警 |
| 连击等级变化 | 切换 BGM 叠层（鼓点渐入） |

### 23.2 视觉反馈增强

- 敌人受击: 根据伤害值显示不同大小的伤害数字
- 玩家受击: 红色径向晕染从受击方向扩散
- 子弹过热（如果加入武器热量机制）: 武器颜色由蓝变红
- 击杀: 粒子爆炸大小/颜色根据敌人类型而变化

### 23.3 屏幕空间特效

- **屏幕抖动分级**: 轻（开火）/ 中（被击）/ 重（Boss 登场/爆炸）（已有，可增强）
- **运动模糊**: 玩家高速移动时边缘模糊
- **色饱和控制**: 低血量时画面去饱和，增加紧张感
- **色差（Chromatic Aberration）**: 强特效时轻微 RGB 分离

### 23.4 背景层升级

当前背景为星星 + 小行星。改为三层结构:
- 远景星: 缓慢漂移的点状星星（视差）
- 中远景: 稀疏星云斑块（静态）
- 近景: 小行星 + 偶尔的星尘粒子（根据玩家速度）

## Phase 24: 多人与合作（Multiplayer & Co-op）

**目标**: 提供本地/在线合作，扩展游戏为"可一起玩"

### 24.1 本地双人分屏（优先）

同屏合作模式:
- 2P 同屏，共享摄像机跟随两玩家中点
- 玩家 1: WASD + 鼠标
- 玩家 2: 方向键 + NumPad0 开火（或手柄）
- 共享科技树/资源，但各自 HP/护盾
- 距离过远时屏幕分割为左右两屏

**实现要点**:
- Core.lua state.players 改为数组（{p1, p2}）
- PlayerCtrl.lua 抽象出 inputSource 参数（keyboard / keyboard2 / gamepad）
- RenderWorld.lua 根据玩家位置重新计算摄像机

### 24.2 在线排行榜升级（已存在基础）

扩展排行榜维度:
- 按游戏模式（赛季/无尽/Boss Rush/限时/弹幕）
- 按难度（标准/困难/虚空）
- 按阵营分别排行
- 全球周榜（重置周期）
- 好友排行

### 24.3 每日挑战优化

当前每日挑战 = 6 modifiers 随机组合。升级:
- 固定每周主题（如 "无护盾周" / "弹幕地狱"）
- 每日挑战专属皮肤奖励
- 可与好友分享"挑战种子"重玩同局

### 24.4 幽灵数据（Ghost Data）

- 玩家每局记录轨迹/按键序列
- 可选择"挑战自己的幽灵"模式
- 排行榜每条记录包含路径数据（极简压缩）

## Phase 25: 移动端与主机适配（Mobile & Console）

**目标**: 让游戏在触屏/手柄上也能流畅游玩

### 25.1 触控操作

| 操作 | 方案 |
|------|------|
| 移动 | 左下虚拟摇杆 |
| 射击 | 默认自动开火（可切换为手动右下按钮） |
| 瞄准 | 手指滑动屏幕右侧（不按摇杆时） |
| 技能 | 右下 1/2/3 圆形按钮 |
| 第二武器 | 屏幕中下方按钮 |
| 科技树/暂停 | 右上角小图标 |

**实现要点**:
- main.lua 新增 touchControls 分支
- RenderUI.lua 新增 drawTouchControls()（仅在移动端/触摸屏）
- PlayerCtrl.lua 新增 inputFromTouch(touchState)

### 25.2 手柄适配

- Xbox / PlayStation 手柄映射
- 移动: 左摇杆

## Phase 26: Mod 生态与编辑器工具（Modding & Tools）

**目标**: 将现有的配置扩展接口升级为可使用的 Mod 系统

### 26.1 Mod 目录结构

游戏启动时扫描 `mods/` 目录，每个 Mod 为一个子目录:
```
mods/
  my_mod/
    mod.json          -- 元数据（名称/作者/版本/依赖）
    enemies.lua       -- 新敌人定义
    bosses.lua        -- 新 Boss 定义
    relics.lua        -- 新遗物
    events.lua        -- 新事件
    assets/           -- 自定义贴图/音效
```

### 26.2 Mod 加载流程

1. 启动时扫描 mods/ 目录
2. 读取 mod.json 元数据
3. 检查依赖/冲突
4. 按优先级顺序加载（官方 mod > 社区 mod）
5. 调用 Data.registerEnemyType / Data.registerBoss / Data.registerRelic 等已存在接口

### 26.3 Mod 管理器 UI

- 主菜单"模组"选项
- 列表展示已安装 mod（名称/版本/作者）
- 启用/禁用切换
- 冲突警告
- 一键打开 mod 目录

### 26.4 Mod 开发文档

在游戏根目录 `mods/_TEMPLATE` 提供:
- 示例 mod.json
- 敌人/遗物/事件示例
- Lua API 文档（Data.Xxx / Core.Xxx / Systems.Xxx）

## 技术风险与缓解

| 风险 | 严重度 | 缓解策略 |
|------|--------|----------|
| 新功能让 Core.lua 过大 | 🔴 高 | 严格按模块拆分: 新建 state/ 子目录放各子系统 |
| Mod 系统兼容性问题 | 🟡 中 | 提供沙盒环境，严格配置校验，记录加载失败堆栈 |

| 移动端性能落差大 | 🟡 中 | 性能降级选项 + 分平台渲染路径 |
| 存档版本迁移失败 | 🟡 中 | 从 v0.8 起严格维护迁移函数，每个版本独立单元测试 |
| 平衡数据混乱 | 🟡 中 | 所有数值集中在 Data.lua，提供配置表导出/导入工具 |
| 代码风格不一致 | 🟢 低 | 建立 CODE_STYLE.md 规范，新增功能严格遵循 |
